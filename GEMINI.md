## Engineering Standards (Principle VIII)
- **Formatting**: Prettier is mandatory.
- **Linting**: ESLint is mandatory.
- **Git Hooks**: Husky and lint-staged must be used to enforce quality.
- **CI/CD**: GitHub Actions must validate all changes on merge to `main`.
- **Coverage**: Minimum 90% test coverage required (100% target).
- **Automation**: Use `package.json` scripts or `Makefile` for common tasks.
